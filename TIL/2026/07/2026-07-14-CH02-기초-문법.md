- **대소문자를 구분함** (`Age`와 `age`는 서로 다른 변수)
- 자료형
- 기본 자료형(int,float,str,bool)
- bool 
- 참(`True`) 또는 거짓(`False`) 딱 두 가지 값만 존재
- 비교 연산자(`>`, `==` 등)의 결과로 자주 만들어짐
- bool( ) 진위값 판단 규 
- 형변환
2. 연산자
- 멤버십 연산자**(in / not in)**
```python
fruits = ["사과", "바나나", "포도"]
print("바나나" in fruits)      # True
print("수박" in fruits)         # False
print("수박" not in fruits)     # True

text = "Python"
# True (문자열 안에 부분 문자열이 있는지도 확인 가능)
print("th" in text)
```

- 식별 연산자 **(is / is not)**
```python
a = [1, 2, 3]
b = [1, 2, 3]
c = a

print(a == b)   # True   (내용물 값이 같음)
print(a is b)   # False  (서로 다른 별개의 리스트)
print(a is c)   # True   (c는 a와 완전히 같은 대상)
```

- 서로 다른 연산자 비교
- 연산자 우선순위
- **산술 → 비교/멤버십/식별 → 논리** 순
- **1순위:** 자료구조 만들기 (리스트 `[]`, 튜플 `()`, 딕셔너리/셋 `{}`)
- **2순위:** 데이터 꺼내기 (인덱싱, 슬라이싱)
- **3순위:** 거듭제곱 (`**`)
- **4순위:** 곱하기, 나누기 계열 (`*`, `/`, `//`, `%`)
- **5순위:** 더하기, 빼기 (`+`,`-`)
- **6순위:** 비트 단위 연산 (이진수 계산)
- **7순위:** 소속 및 동일성 비교 (`in`, `not in`, `is`, `is not`, 그리고 크기 비교)
- **8순위:** 논리 연산 (`not`, `and`, `or` - 논리 연산 안에서도 not -> and -> or 순서)
- **9순위:** 한 줄짜리 조건문 (`if`, `else` 표현식)
- **10순위:** 이름 없는 임시 함수 만들기 (`lambda`)

1. 입출력 옵션
- sep : 값들 사이에 구분자를 원하는 문자로 변경(default: 공백)
- end : 출력이 끝난 뒤 자동으로 줄바꿈되는 부분을 다른 문자로 변경(default: \n)
```python
print('apple','banana','plum',sep='/')
#결과값: apple/banana/plum

print('hello',end="**")
print('world')
#결과값: hello**world
```

- 입력 받은 값은 항상 문자열임. **→ 숫자로 입력받아도 문자열로 저장됨**
- 여러값 입력받는 방법 — input( )/split( )
```python
num1=int(input('첫번째 숫자를 입력하세요: '))
num2=int(input('두번째 숫자를 입력하세요: '))
print('합계: ',num1+num2)

nums=input('두 숫자를 공백으로 구분해서 입력하세요: ')
a,b=nums.split()
print(int(a)+int(b))
```

- `input()` 값을 형변환 없이 더하면 문자열 이어붙이기가 됨 (`"3"+"5"` → `"35"` )

## 💭 오늘의 회고

1. **배운 점**
- **값 반환(Return) vs 화면 출력(Output)의 차이**
- `type(x)`: 컴퓨터 내부적으로 자료형을 알아내기만 하고 조용히 넘어감.
- `print(type(x))`: 알아낸 자료형 결과를 화면에 명시적으로 출력함. 사용자가 눈으로 확인하려면 반드시 `print()`로 감싸야 함.
- **f-string (포맷된 문자열 리터럴)**
- 문자열 앞에 `f`를 붙이고 중괄호 `{}` 안에 변수명이나 연산식을 넣으면 코드가 압도적으로 깔끔해짐.
- *예시:* `print(f"내년 나이는 {int(age) + 1}살입니다.")`
- **연산자 우선순위 (앞에 있을수록 먼저 계산됨)**
- 1순위: 구조 세팅 (`[]`, `()`, `{}`) / 데이터 꺼내기 (인덱싱, 슬라이싱)
- 2순위: 거듭제곱 ()
- 3순위: 산술 연산 (, `/`, `//`, `%` ➔ `+`, )
- 4순위: 소속/논리 연산 (`in`, `not in` ➔ `and` ➔ `or`)
- 마지막: 조건문, 람다 함수
- `print()`**의 **`end`** 옵션**
- 출력의 끝을 장식하는 문자를 지정. 기본값은 줄바꿈(`\n`)이지만, `end=""`로 설정하면 줄바꿈 없이 다음 출력물과 딱 붙여서 출력할 수 있음.
- **파이썬만의 부등호 연속 사용**
- 수학 기호처럼 `18.5 <= BMI < 23` 형태로 연속된 부등호를 사용할 수 있어 `and` 연산자 없이도 직관적인 조건문 작성이 가능함.


2. **어려운 점/개선할 점**
- `input().split()`으로 받은 값을 바로 `a + b` 하면 문자열끼리 연결됨 ("10"+"20"="1020"). 숫자로 쓰려면 `map(int, input().split())`을 사용해야 함.
- `map(str, int, str, ...)`처럼 여러 자료형을 한 번에 다르게 지정하려고 시도함. `map()`의 첫 번째 자리에는 단 하나의 자료형/함수만 들어갈 수 있음.
- BMI 계산 시 `kg // m * m`으로 작성함.
→ `//`는 소수점을 버리는 '몫' 기호이므로 정확한 계산에는 `/`를 써야 함. 또한, 괄호가 없으면 앞에서부터 계산되어 `(kg / m) * m`이 됨. 정확히 키의 제곱으로 나누려면 `kg / (m**2)`로 거듭제곱을 활용해야 함.
- `input()`을 `while`문 바깥에 딱 한 번만 쓰고 루프 안에서 갱신하지 않거나, 무조건 발동하는 `break`를 넣음.
→  `while`문은 조건이 계속 참이면 무한 루프에 빠지므로, 루프 내부에서 변수값을 바꾸는 작업(예: 새로운 `input()` 받기, 카운트 증가)이 반드시 들어가야 함. 목표 횟수를 모르는 누적 계산은 `while True:`와 `if 조건: break` 조합이 가장 깔끔함.
