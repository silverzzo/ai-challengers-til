## 📝 오늘 배운 내용 요약

1. ** List Comprehension**
- if문 추가
[표현식 for 변수 in 반복대상 if 조건]
```python
# 1~30 중 3의 배수만

num=[i for i in range(1,31) if i%3==0]
print(num)

#[3, 6, 9, 12, 15, 18, 21, 24, 27, 30]
```

- if-else 문 추가
[조건참일때값 if 조건 else 조건거짓일때값 for 변수 in 반복대상]
```python
# 60점 이상이면 "합격", 아니면 "불합격"으로 변환
scores = [85, 45, 92, 58, 76]

score=['합격' if s>=60 else '불합격' for s in scores]
print(score)

#['합격', '불합격', '합격', '불합격', '합격']
```

- 중첩 반복
[조건참일때값 if 조건 else 조건거짓일때값 for 변수 in 반복대상]
```python
# 구구단 2~3단을 한 번에 리스트로

result=[i*j for i in range(2,4) for j in range(1,10)]
print(result)

#[2, 4, 6, 8, 10, 12, 14, 16, 18, 3, 6, 9, 12, 15, 18, 21, 24, 27]
```

## 💭 오늘의 회고

1. **배운 점**
- `map()`**과 **`filter()`**의 차이**: `filter()`는 조건에 맞는 값만 걸러내고, `map()`은 모든 값을 원하는 형태로 변환한다는 차이를 이해함. `lambda x: ...`의 `x`는 전달되는 값을 받는 매개변수라는 점도 익힘.
- **함수와 메서드의 차이**: 함수는 독립적으로 호출되고, 메서드는 객체에 소속되어 호출된다는 개념을 이해함.
- `sort()`**와 **`sorted()`**의 차이**: `sort()`는 원본 리스트를 직접 변경하고, `sorted()`는 원본은 유지한 채 정렬된 새로운 리스트를 반환한다는 차이를 이해함.
- `range`** 객체의 동작 방식**: `[range(1, 11)]`처럼 리스트로 감싸면 `range` 객체 자체가 하나의 요소로 저장된다는 점을 이해함.
- **내장 함수와 변수명 충돌**: `sum`을 변수명으로 사용하면 내장 함수 `sum()`을 사용할 수 없게 된다는 점을 알게 됨.
- **다중 컴프리헨션 작성 순서**: 이중 `for`문을 컴프리헨션으로 바꿀 때는 기존 `for`문의 순서를 그대로 유지하고, 최종적으로 저장할 값을 맨 앞에 작성한다는 규칙을 이해함. 소괄호만 사용하는 형태는 제너레이터 표현식이라는 것도 함께 익힘.
2. **어려운 점/개선할 점**
- `map()`**·**`filter()`**·**`lambda`** 문법**: 람다식 문법이 아직 익숙하지 않음. 직접 여러 번 작성해 보면서 형태를 익힐 필요가 있음.
- `range`** 객체와 리스트 구조**: 리스트로 감쌌을 때 자료구조가 어떻게 바뀌는지 헷갈렸음. `type()`이나 출력 결과를 함께 확인하는 습관이 필요함.
- **내장 함수 이름 사용**: `sum`처럼 내장 함수 이름을 변수로 사용하는 실수가 있었음. `total`, `total_sum`처럼 의미 있는 변수명을 사용하는 습관을 들여야 함.
- **다중 컴프리헨션 순서**: `for`문이 여러 개 들어가면 작성 순서가 헷갈렸음. 먼저 일반 이중 `for`문으로 작성한 뒤 컴프리헨션으로 변환하는 연습이 필요함.
