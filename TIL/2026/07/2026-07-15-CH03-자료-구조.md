1. 문자열 처리
- 주로 쓰는 문자열 메소드
- 문자열은 인덱싱/슬라이싱이 가능하지만 수정은 불가능(불변)


2. f-string
```python
name = "홍길동"
age = 25

print(f"이름:{name}, 나이:{age}")
```

- 콤마 나열 vs `+` 연결 vs f-string —  f-string을 기본으로 사용하는 것을 권장
```python
name, age = "홍길동", 25

# 방법 A: print()에 콤마로 나열
print("이름:", name, "나이:", age)

# 방법 B: 문자열 + 연결 (숫자는 반드시 str()로 변환해야 함)
print("이름: " + name + ", 나이: " + str(age))

# 방법 C: f-string (가장 간결하고 읽기 쉬움)
print(f"이름:{name}, 나이:{age}")
```

3. 리스트[ ]
- 서로 다른 자료형을 함께 담을 수 O. 
- 변경가능(생성 후에도 요소를 추가/삭제/수정 가능)
- 주요 리스트 메소드
- 리스트에 값 추가 - append( ) vs extend ( )
4. 튜플( )
- 순서는 있지만 변경이 불가능. ⇒ dict 키로 사용 가능
- 리스트와 달리 **생성 후 요소를 변경할 수 없음(immutable)**
- 요소가 하나뿐인 튜플은 반드시 ‘,’가 필요
```python
not_tuple=(5)
real_tuple=(5,)

print(not_tuple)  #5    <class 'int'>
print(real_tuple) #(5,) <class 'tuple'>
```

- 주요 튜플 메서드
```python
t = (1, 2, 2, 3, 2)

print(t.count(2))    # 3   (2가 몇 번 나오는지)
print(t.index(3))     # 3   (값 3의 위치)
```

-  튜플 언패킹
```python
point = (3, 5)
x, y = point            # 언패킹: x=3, y=5
print(x, y)

name, age, city = ("홍길동", 25, "서울")
print(name, age, city)
```

5. 딕셔너리 { 키 : 값 }
- 순서 없음
- 키는 고유해야 함(중복 불가), 값은 중복 가능
- 주요 딕셔너리 메서드
```python
person=({'name':'eun','age':25})

print(person.keys())      #dict_keys(['name', 'age'])
print(person.values())    #dict_values(['eun', 25])
print(person.items()))    #dict_items([('name', 'eun'), ('age', 25)])

age=person.pop('age')
print(age,person)         #25 {'name': 'eun'}

person.update({'age':30,'city':'busan'})
print(person)             #{'name': 'eun', 'age': 30, 'city': 'busan'}

person.setdefault('job','none')
print(person)             #{'name': 'eun', 'age': 30, 'city': 'busan', 'job': 'none'}
```

- 조회 방식 - `dict[key]`** vs **`.get(key)`** **
- 키 존재 여부 확인 - in 연산자
```python
person = {"name": "홍길동", "age": 25}

print('name'in person)  #True
print('job'in person)   #False
```

6. Set
- 중복 자동으로 제거 → 리스트의 중복값 제거할 때 셋으로 변환
- 순서 없음 → 인덱싱•슬라이싱 불가능
- 빈 셋 생성할 때는 set( )
```python
l=[]            #빈 리스트 :조건에 맞는 데이터만 모을 때
empty_dict={}   #빈dict : 항목별로 분류하거나 개수를 셀 때
empty_set=set() #빈셋 :중복 없이 데이터를 수집할 때
```

- 셋에 값 추가
```python
fruits={'apple','banana'}
print(fruits) #{'banana', 'apple'}

fruits.add('plum')
print(fruits) #{'banana', 'plum', 'apple'}

fruits.add('apple')
print(fruits) #{'banana', 'plum', 'apple'} -> 중복 허용 안하기 때문에 apple이 이미 있어서 한번만 나옴
```

- 삭제 - remove( ) / discard( ) :**없는 값을 삭제하려 할 때** 동작이 다름
- 집합 연산,메서
```python
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

# 교집합: 둘 다 포함된 값
print(a&b)
print(a.intersection(b))

# 합집합: 둘 중 하나라도 포함된 값
print(a|b)
print(a.union(b)) 

# 차집합: a에는 있지만 b에는 없는 값
print(a-b)
print(b-a)
print(a.difference(b))             
print(b.difference(a))

# 대칭차집합: 겹치지 않는 값들만
print(a^b)
print(a.symmetric_difference(b))
```

7. zip( )과 enumerate( )
-  enumerate( ) : 반복하면서 인덱스도 함께 얻기
- zip( ) : 여러 목록을 짝지어 동시에 반복하기
- 길이가 다른 목록을 zip( )하면 짧은 쪽으로 맞춰
8. List Comprehension
- [결과값 for 변수 in 반복범위]
```python
#기존
s=[]

for i in range(1,6):
  s.append(i**2)
print(s)

#리스트 컴프리헨션
s=[i**2 for i in range(1,6)]
print(s)
```

## 💭 오늘의 회고

1. **배운 점**
- **딕셔너리와 셋(Set)의 동작 원리**
- 딕셔너리를 `for`문으로 돌리면 기본적으로 key가 반환된다는 것을 이해
- value나 (key, value) 쌍이 필요하면 `.values()`, `.items()`를 명시적으로 써야 함.
- 없는 키에 바로 접근하면 `KeyError`가 나므로, `.get()`을 활용해 안전하게 값을 꺼내야 함.
- `{}`는 빈 딕셔너리로 취급되기 때문에 빈 셋은 반드시 `set()`으로 선언해야 함.
- **튜플 언패킹의 구조 이해**
- `zip()`과 `enumerate()`를 함께 쓰면 `(인덱스, (값1, 값2))` 형태의 이중 튜플 구조가 만들어짐.
- 이 구조에 맞춰 `for`` idx, (name, score)`` in ...`처럼 괄호로 묶어 언패킹해야 에러 없이 순회 가능.
- **리스트 메서드의 반환값 특성**
- `sort()`, `append()` 같은 원본을 직접 수정하는 메서드는 `None`을 반환
⇒ `list = l.sort()`처럼 재할당하면 안 되고, 메서드 호출 자체로 원본이 바뀜.
- 리스트 컴프리헨션은 그 자체로 append 기능을 내장하고 있어서, 대괄호 안에는 저장할 값(수식)만 써야 함.
- `zip()` 같은 함수가 즉시 값을 안 보여주고 객체 주소를 반환하는 이유
- 대용량 데이터를 한 번에 메모리에 올리지 않고 필요할 때마다 하나씩 꺼내주는 **지연 평가(Lazy Evaluation)** 방식 때문
- **예약어/내장 함수 이름 덮어쓰기(Shadowing) 주의**
- 변수명을 `set`, `list`, `dict` 등으로 지으면 파이썬 내장 함수가 덮어씌워져서 `TypeError: 'set' object is not callable` 같은 에러가 발생
- `del set`로 덮어씌워진 변수를 지워 원래 기능을 복구가능.


2. **어려운 점/개선할 점**
- **딕셔너리 값 접근 습관**: 
`.get()` 대신 대괄호로 바로 접근하려는 습관이 남아있음. 중첩 딕셔너리(`students['kim']['job']`)처럼 구조가 깊어지면 에러 위험 여전히 큼. 
→ 중첩 구조에서 `.get()` 체이닝을 기본값으로 쓰는 연습 필요함.
- **튜플 언패킹 시 괄호 구조 파악이 느림**: 
`zip()`, `enumerate()`가 겹칠 때 실제 데이터 구조를 바로 그려내지 못해 `ValueError`를 반복적으로 마주침. 
→ `print()`로 반환 구조를 먼저 찍어보고 언패킹하는 습관 들이기.
- **변수명과 출력 변수 불일치**: 
`for`문에서 개별 변수로 나눠 받아놓고 `print`문에서는 원본 리스트 이름이나 정의 안 된 변수를 써서 `NameError` 발생시킴. 
→ 변수 선언 직후 어떤 이름을 써야 하는지 바로 확인하는 버릇 들이기.
- **컴프리헨션 안에서 부수효과(append) 쓰려는 습관**: 
리스트 컴프리헨션의 본질(값을 만들어 담는 것)을 아직 완전히 체화하지 못함. `append()`를 끼워 넣으려다 막히는 패턴 반복됨.
→ "컴프리헨션 = 결과값을 만드는 식"이라는 원칙 먼저 떠올리기.
- **예약어/내장함수 네이밍 습관 미정착**: 
`set`, `list` 같은 이름을 변수로 무심코 쓰는 습관 남아있음. 
→ `word`, `my_dict`처럼 구체적인 이름을 쓰는 걸 규칙으로 만들기.
