- 통계함수
```python
prices = np.array([189.5, 191.2, 187.8, 193.4, 195.9])

print(prices.mean())    # 평균: 191.56
print(prices.sum())      # 합계
print(prices.max())      # 최댓값
print(prices.min())      # 최솟값
print(prices.std())      # 표준편차 (변동성 측정에 활용)
```

```python
# "값"이 아니라 "위치"가 궁금할 때 — argmax / argmin
prices = np.array([189.5, 191.2, 187.8, 193.4, 195.9])

print(prices.argmax())   # 4  → 5일차(인덱스 4)가 최고가
print(prices.argmin())   # 2  → 3일차가 최저가

```

—  `max()`는 "얼마나", `argmax()`는 "언제/어디서"에 답함.

- 정렬과 모양 바꾸기 - `np.sort()`, `reshape()`

prices = np.array([189.5, 191.2, 187.8, 193.4, 195.9])

## 💭 오늘의 회고

1. **배운 점**
- `np.array()`로 배열 생성, `shape`/`ndim`/`dtype`/`size`로 속성 확인 — 배열은 **동일 자료형**만 저장
- 인덱싱/슬라이싱: 1차원은 `arr[i]`, 2차원은 `arr[행, 열]` (`:`는 "전부")
- 배열의 `+`, 는 리스트와 달리 실제 수학 연산 (같은 위치끼리 계산) — 환율, 할인, 수익률이 한 줄로
- 통계: `mean()`/`sum()`/`max()`/`min()`/`std()` + 위치를 찾는 `argmax()`/`argmin()`
- **axis=0은 ↓열별, axis=1은 →행별** — 결과 개수로 검산하는 습관
- 불리언 인덱싱 `arr[arr > 값]` = 조건으로 골라내기 — 다음 시간 Pandas 필터링의 원리
- `np.where(조건, 참값, 거짓값)` = 조건에 따라 값 바꾸기
- `np.sort()` 정렬, `reshape()` 모양 변경 (요소 개수 유지)
2. **어려운 점/개선할 점**
- **리스트 vs NumPy 배열 인덱싱 방식 혼동**: `[1, 2]`와 `[1][2]`가 결과는 같아 보여도 내부 동작이 다르다는 걸 직관적으로 구분하지 못함. → 다차원 인덱싱은 `arr[행, 열]` 형태를 기본으로 쓰는 습관 필요함.
- **괄호 기호별 역할 혼동**: f-string 안에서 `{days{best}}`처럼 중괄호를 겹쳐 써서 `SyntaxError`를 겪음. 값을 꺼내는 대괄호 `[]`와 문자열 포매팅용 중괄호 `{}`를 헷갈림. → f-string 안에 인덱싱/함수 호출을 넣을 땐 `{days[best]}`처럼 괄호 종류를 명확히 구분해서 쓰기.
- **중첩된 코드 해석**: `days[cafe.argmax()]`처럼 인덱싱 안에 함수 호출이 통째로 들어간 구조:  중첩 코드는 가장 안쪽 함수/연산부터 계산 → 그 결과가 바깥 인덱싱에 사용됨
- **prices[1][2] vs prices[1, 2] 비효율성 인지 부족**: 대괄호를 두 번 쓰면 임시 배열이 한 번 더 생성되어 비효율적이라는 걸 모르고 습관적으로 리스트 방식(`[1][2]`)을 쓰려 함. → NumPy 배열에서는 콤마 인덱싱을 기본값으로 쓰는 습관 필요함.
- **len(배열[조건])과 (조건).sum()의 내부 차이 **: 둘 다 같은 결과(개수)를 내지만 메모리/속도 면에서 다름. (**실제 필터링된 값 자체가 필요할 때**는 `배열[조건]`을 쓰는 게 맞고, **개수만 필요할 때**는 `.sum()`이 더 효율적)
